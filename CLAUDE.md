## テーマと仮説

### 検証したい仮説
> 何らかの要因で最もストレスがかかっている時（大負け、仕事ストレス、借金プレイ）に、
> 大逆転の可能性を脳が認知した瞬間、最強のドーパミンが出るのではないか。

ユーザー自身が15年間ギャンブル依存症だった経験の「答え合わせ」をしたい、というのが本テーマの動機。

### 検証アプローチ
ハッカソン配布のベースコード（`reference/` にある 2D マルチエージェントLLMシミュレーション）を改造し、
**パチスロ挙動を忠実に再現した環境**でLLMエージェントを動かす。
エージェントには**数値データのみ**を与え、「興奮しろ」等の指示は一切与えない。
LLM が出力する `reasoning` / `message` に "興奮" 相当の言語が創発するかを観察する。

### 改造の対応関係（叩き台）
| ベースコード | パチスロ版 |
|------------|-----------|
| `places`（bar等の場所） | パチスロ台（各台ごとに期待値・設定・履歴） |
| `fires`（火事イベント） | ストレス事象（大負け・仕事ストレス・借金）or 大逆転の兆し（リーチ・確変演出） |
| 占有率・距離 | 残金・負け額・ボーナス確率・連敗数 |
| 観察指標 | `reasoning` / `message` に現れる感情語彙、行動継続の頻度 |

## ベースコードの要点

配布コード: `reference/2d-multi-places-simulation-on-fire-public` 相当（直接 `reference/` 配下）

- ollama 駆動のローカルLLM
- デフォルト: 20エージェント × 100ステップ × 2回推論 = 約4000回の LLM 呼び出し
- 各エージェントは毎ステップ `message`（会話）→ `action`（移動）を出力
- エージェントには**定量データのみ**を与え、定性ラベルは一切含めない設計哲学
- 場所（places）は収容上限と位置を持つ。火事イベント（fires）は `start_step` で発生

主要ファイル（`reference/` 内）:
- `main.py` — エントリポイント
- `simulation.py` — シミュレーションループ
- `agent.py` — エージェント本体（LLMプロンプト生成）
- `ollama_client.py` — ollama API クライアント
- `config.yaml` — パラメータ定義
- `visualization.py` — 可視化
- `visualization/viewer.html` — ブラウザで結果閲覧
- `visualization/generate_video.py` — MP4動画生成
