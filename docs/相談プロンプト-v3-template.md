# ChatGPT Pro 相談プロンプト v3（進め方と方向性を決める）

## 使い方

1. Deep Research（`docs/deep-research-prompt.md`）の結果が返ってきたら、本ファイルの
   `[DR 結果をここに貼る]` という箇所を DR 結果で置き換える
2. 仕上がった全文を ChatGPT Pro にコピペして投げる
3. ChatGPT Pro の回答で路線と実行プランが決まったら、CLAUDE.md を最終版に更新して実装開始

このテンプレの目的は、前回 v2（「最適な1つの構成を出せ」型）とは違って、
**「方向性と進め方を私と一緒に決めるための相談」** にすること。
最終判断はユーザーに戻す構造で設計している。

# ハッカソン企画「パチンコホール群像のLLM心理シミュレーション」— 進め方と方向性の決定相談

## 0. この相談のゴール

実装計画を1つ提示するのではなく、**私がこれから進める方向性を決めるための選択肢・限界・判断軸を提示してほしい**。具体的には:

1. 私の希望（100 ペルソナ・3 機種・群像劇）が実際にどこまで可能かのリアリティチェック
2. 「ここまでなら容易」「これ以上は厳しい」という具体的ライン
3. 3 つ程度の方向性の選択肢（それぞれの成果物イメージ付き）
4. 推奨路線 + 私が判断するための軸
5. ローカル LLM 初心者の私が試行錯誤モードに入るまでの最短ルート
6. 方向性決定後の 3 週間実行プラン

「最適解を1つ押しつける」のではなく、**「私がこの 3 週間をどう使うか決められる材料を揃える」**ことを目指してほしい。

## 1. ハッカソンの提出要件（絶対制約）

「シンギュラボ」コミュニティのハッカソン（締切 **2026-05-07**、残り約3週間）。

### 配布物
主催側から `2d-multi-places-simulation-on-fire-public` というベースコードが配布されている。
これを改造または拡張して成果物にする。

### 提出物（3点）
1. ソースコード（GitHub が理想）
2. **シミュレーション結果の可視化動画**
3. 簡単な解説文章

### 配布コードの設計哲学（絶対に守るべき）
- **Ollama** でローカル LLM を駆動（`localhost:11434`）
- エージェントには **定量データのみ** を与える（「危険」「嬉しい」等の定性ラベルは一切含めない）
- 各ステップで各エージェントが `message`（会話）と `action`（行動）を LLM から生成
- 場所（`places`）は位置と収容上限、イベント（`fires`）は `start_step` で発生
- デフォルトは 20 エージェント × 100 ステップ × 2 推論/ステップ = 約 4000 LLM call
- 出力: frame画像、`messages.jsonl`、`memory_reasoning.jsonl`、シミュレーションログ
- 可視化ツール（viewer.html、generate_video.py）も同梱

### 主催者コメント要約
- 「ollama や python に慣れていない人には難しい」「ある程度のスペックの PC 必須」
- 「Claude / ChatGPT に読み込ませて相談するのを推奨」
- 「参加者同士のコラボも推奨」
- **評価軸は明示されていない**。提出物3点の完成度＋企画の独自性が鍵と推測される

## 2. 企画の全体像

### 最優先の観察目的
1. **最も興奮する瞬間の観測**（ドーパミン相当 proxy のピーク条件の特定）
2. **最も不幸になる瞬間の観測**（絶望・後悔・loss chasing のピーク）
3. 自分が知らない社会的背景の人の内面を知る（共感的動機）
4. **AI が人間のような感情や行動をどう示すかを観察して遊ぶ**（探索的目的）

### 独自仮説（企画の核心）
- ユーザーは 15 年間ギャンブル依存症だった実体験者
- 体感仮説: **「最大ストレス × 大逆転認知の瞬間」に最強のドーパミンが出る**
- 強化版: **「2010年代の爆裂 ART 機」は「2000年代の爆裂 AT 機」より中毒性の質が違う**
  （2010s: 一度連チャンに入ると半日〜1日止まらない／2000s: 出ても1-2時間で終わる）
  - この**世代差の機序をエージェント挙動の差分として観察する**ことが企画の目玉

### 再現したい要素

**(A) 機種パターン 3世代**

| 機種 | 世代 | 特徴 |
|------|------|------|
| 爆裂 AT 機（ミリオンゴッド系） | 2000年代 | 一撃万枚、分散極大、短時間型 |
| 4.5号機（北斗の拳系） | 2000年代後半 | AT/ART 性能高い、波大 |
| 爆裂 ART 機 | 2010年代 | **連チャンが長時間継続（中毒性の本体）** |

**(B) ペルソナ: 30〜100 人規模の群像**

初期案は 5 ペルソナ。ユーザーの強い希望で **30〜100 人規模の多様な群像**を並置し、
**自分が知るリアルなパチンコホールの空間を再現したい**。
「色々な人間に色々な感情が起こる様を観測したい」「AI が人間っぽくどう振る舞うか見たい」
という**探索的動機**が核心。

中核の 5 人:
| # | ペルソナ | 観察視点 |
|---|---------|---------|
| a | 借金を抱えた学生 | 返済圧 × 若さの無計画 |
| b | 夫に隠れて打つ主婦 | 罪悪感 × 家計ストレス × 孤独 |
| c | 依存症サラリーマン | 長期依存 × 自己嫌悪 × 稼ぎあり |
| d | 興味はあるが金なしの大学生 | 好奇心主導、損失ナイーブ |
| e | バイト代10万でルンルンの学生 | **健常者が熱くなる過程の観察（変容観察）** |

残り 25〜95 人は、パチンコホールに実在する多様性を再現:
年齢（20代〜70代）、性別、職業、依存度、所持金、来店動機、心理状態などで分散。

**(C) 観察アウトカム**
- 覚醒語彙の自然発生頻度（「まだいける」「熱い」「来る」「取り返せる」等）
- 不幸語彙の頻度（「終わった」「絶望」「後悔」等）
- `continue` / `quit` の意思決定
- 賭け額・プレイ継続時間の escalation
- ペルソナ間の感情表現の質的差
- 機種間の継続ステップ数の差
- 群像全体として「ホールの熱量」のピーク

### スペック活用の希望
ユーザーは M4 Max 64GB を活かしたい。「スペックを使わないプラン」は避けたい。
ただし 3週間内で安定して回ることが絶対条件。

## 3. 実装パートナーの文脈

ユーザーは以下の AI アシスタント環境で実装を進めている。**この前提で実行計画を立てると
大幅にスコープを広げられる**:

- **Claude Code (Claude Opus 4.7, 1M context)** をメインのコーディングパートナーとして使用
  - コード生成・リファクタリング・プロンプト合成・設計を任せられる
  - 1M context なので配布コード全体 + 論文 + ドキュメントを全部読める
  - スキル・エージェント・プランモードで複雑なワークフローを自動化できる
  - 100 ペルソナの設定ファイルを Claude Code に一括生成させる前提で設計可能
  - プロンプト探索（A/B テスト）も自動化できる
- **ChatGPT Pro（Deep Research 使用中、あなた）** を設計相談・最新情報収集に使用
- ユーザー自身は Python を読めるが、ローカル LLM は未経験

つまり「人間が手書きする分量」はほぼゼロで良い。複雑な設計を指示すれば Claude Code が
実装する。**実行計画を立てる時は、「人間の実装負荷」ではなく「Claude Code に
指示するコストと検証コスト」を最適化してほしい。**

## 4. すでに確定した技術前提

前回の ChatGPT Pro 相談（DR の前の初回相談）で以下が確定:

- 「巨大モデル一点賭け」ではなく「中型 × non-thinking × 短文出力 × 最後だけ上位」戦略
- `message` と `action` を別モデルに分けるのが有効
- 仮説精密化: 「stress 単独」ではなく「**stress × comeback-cue の相互作用**」
- 撤退ゲート: **4/21 stock demo 動作 / 4/25 pilot 通過 / 4/29 2×2 ログ / 5/02 図+主張**
- 縮退順序: action 捨てる → message-only → ルールベース + LLM 解釈のみ
- Ollama 罠: thinking 切り忘れ / context 積みすぎ / keep_alive / num_predict / GPU fallback / server overload
- 「定量 only ≠ 無名ベクトル」（`stress_index=0.83` のように名前付きで渡す）

現状:
- Ollama 0.21.0 インストール済み
- `qwen3:4b-instruct-2507-q4_K_M`（2.5GB）pull 済み、単発対話で日本語出力確認済み
- `PROCESSOR=100% GPU` 確認済み

### C. 3 つの方向性の選択肢

以下のような 3 路線を、**成果物イメージ付き**で並べてほしい。
抽象的な比較ではなく、「この路線を選んだら、動画はこう見える、解説文はこう書ける」という
**具体的なイメージ**が湧くレベルで。

案（これに限定せず、より良い路線があれば差し替え可）:

- 🌆 **群像劇路線**: 50-100 ペルソナ、3 機種、ホール全景
- 🔬 **少数密観察路線**: 5-10 ペルソナ、1-2 機種、深堀り
- 🎭 **ハイブリッド路線**: 密観察 10-20 + 群像 30-50、バランス型

各路線で以下を明示:
- 具体的なスケール（agent 数 × step 数 × モデル）
- 1 run 所要時間、3 週間の総実行時間
- 成果物（動画）の見え方（冒頭、中盤、クライマックス）
- 解説文の核メッセージ
- 難所と詰まるポイント
- その路線が刺さる / 刺さらない条件

### D. 推奨路線と判断軸

ChatGPT Pro の視点で 1 つを推薦してほしい。ただし**条件付き推薦**で:
- 「あなたが仮説の刺さりやすさを最優先するなら推奨」
- 「あなたが動画の見栄えを最優先するなら別路線」
- 「あなたが初心者として学びを最大化したいならまた別路線」

のように、**私が最終判断するための判断軸**を明示してほしい。

### E. 進め方のアドバイス（学習戦略）

ローカル LLM 未経験者としての進め方:

- 撤退ゲート（4/21 / 4/25 / 4/29 / 5/02）で何を確認し、何を削るか

**「これができれば後は自分で遊べる」** の具体的チェックリスト。
「あなたが自走できるのは、以下が揃った時」という形で書いてほしい。
各項目に「所要時間の目安」と「詰まりやすい罠」を添えてほしい。

### G. 方向性決定後の 3 週間実行プラン

私が路線 X（C/D で推奨されたもの）を選んだと仮定して:
- 3 週間の週次プラン（Week 1 / Week 2 / Week 3）
- 各週の目標と撤退ゲート
- **Claude Code に何を任せるかの分担表**（実装、プロンプト生成、ペルソナ作成、ログ分析など）

### H. リスクと縮退順序

この企画で最も詰まる可能性が高いポイントと、詰まった時の縮退順序。
特に:

- ペルソナ崩壊・同質化が起きた場合の対処
- 時間切れが見えた時、何を捨てると全体が守れるか
- 完全に詰まった時の「最低限これさえ出せば提出可能」ライン

### I. ハッカソン提出物の形

- 動画の構成案（群像劇 or 密観察、どちらでも）で審査に刺さる構成
- 解説文の核心メッセージ（独自仮説 + 群像シミュレーションとしての価値）
- GitHub README の構成
- **動画 + 解説文を作る時間を確保するための逆算**（最後の 2-3 日で何をする？）

## 9. 判断軸（最適性の定義）

「最適」を以下の重みで判断してほしい:

1. **仮説の刺さりやすさ**（独自仮説と群像劇が動画1本で伝わるか）= 35%
2. **観察の豊かさ**（多ペルソナ × 3機種の差分と感情の多様性が見える）= 30%
3. **3 週間での実現可能性**（撤退ゲート全通過、Claude Code 実装前提）= 25%
4. **M4 Max 64GB スペックの活用度**（この環境だからできる規模）= 10%

迷ったら (1) > (3) > (2) > (4) の順で優先してほしい。
「安全だが平凡」より「攻めてるが通る」を選んでほしい。ユーザーは実体験者なので、
表層的なシミュレーションより**本質を刺す設計**を求めている。

ただし、**私は初心者であり、試行錯誤できる段階までの効率**も重視してほしい。
「攻めた最終形」と「そこに到達するための現実的なステップ」の両方が必要。
