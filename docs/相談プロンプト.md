# ローカルLLMの機種選定＆進め方の相談

## 相談の背景
「シンギュラボ」というコミュニティのハッカソンに参加する。課題は、2D マルチエージェントLLMシミュレーションのベースコードを改造し、自分の興味テーマで実験して成果物を提出すること。締切は **2026-05-07**（残り約3週間）。

## ベースコードの概要
- 2D グリッド空間上で、複数の LLM エージェントが会話・移動・記憶を通じて創発的行動を見せるシミュレーション
- **ollama** でローカルLLMを駆動する構成
- エージェントには **定量データのみ** を与え、「危険」「嬉しい」等の定性ラベルは一切与えない
- 各ステップで各エージェントが `message` / `memory` / `action` をLLMから生成
- デモの config では `gpt-oss:20b` または `qwen3:4b-instruct-2507-q4_K_M` が指定されている
- エージェント数は 20（デフォルト）、duration 100 ステップ
- **毎ステップ × 全エージェント × 2回のLLM推論（メッセージ決定 + 行動決定）** が走る → 推論回数がかなり多い（100ステップで約4000回）

## 取り組みたいテーマ
**パチスロの挙動を忠実に再現し、ギャンブル依存症時の脳の反応を検証する。**

仮説:「何らかの要因で最大ストレスが掛かっている時（大負け、仕事のストレス、借金プレイなど）に、大逆転の可能性を脳が認知した瞬間、最強のドーパミンが出るのではないか」

これを、ベースコードの `places`（場所）を**パチスロ台**に、`fires`（イベント）を**ストレス事象 or 大逆転の兆し**に置き換え、エージェントの reasoning/message に "興奮" が言語として創発するかを観察することで検証したい。

## 聞きたいこと

### Q1. モデル選定
私の環境・スキル・やりたいこと（マルチエージェント×多ステップ推論で "内面の言語化" を観察）を踏まえると、どのローカルLLMが候補になる？ 以下の観点でトレードオフを教えてほしい：

- **推論速度**（20エージェント × 100ステップ × 2回推論 = 4000回の推論を現実的な時間で回せるか）
- **日本語能力**（reasoning/message に日本語で「興奮」や「焦り」が出るか）
- **推論の質**（定量データから定性的な感情を言語化する能力）
- **セットアップの手軽さ**（初学者がハマりにくいか）
- **メモリ消費**（M4 Max 64GB で本当に動くか、余裕はあるか、70B 量子化モデルも現実的か）

できれば「軽量だけど物足りない寄り」「バランス型」「重いけど高品質」の3〜4候補でラダー化して、私の場合のおすすめ出発点を示してほしい。

### Q2. 段階的な進め方
3週間しかないので、以下の順序で進めるのが妥当か、もっと良い進め方があるか教えてほしい：

1. ollama インストール → 軽量モデルでまずデモをそのまま動かす
2. 動いたらモデルを差し替えて品質と速度のバランスを探る
3. config.yaml の places/fires をパチスロ版に改造（コード本体はまだ触らない）
4. 仮説検証用の比較条件（ストレス有無 × 大逆転有無）を設計
5. 実験実行 → 可視化動画生成 → 解説文執筆

### Q3. 初学者が引っかかりやすい罠
ローカルLLM未経験者が ollama + マルチエージェントでやりがちなハマりどころ（プロンプト設計、コンテキスト長、温度設定、推論ループの詰まり等）を事前に教えてほしい。

### Q4. 「やっぱり無理だった」判定の基準
残り3週間で進捗が芳しくない場合、どの段階で「ローカルLLMは諦めて別アプローチ（例: 強化学習のみ、ルールベース+軽量モデル等）に切り替える」判断をすべきか、チェックポイントを示してほしい。

### Q5. パチスロ再現で気をつけるべき点
パチスロの挙動（期待値、設定判別、連荘、ハマり、リーチ演出、確変突入、確率収束）を数値モデルで再現する上で、**仮説検証のノイズにならないよう最小限に絞るべきか／リアルに作り込むべきか** 助言がほしい。3週間という時間制約の中で、どこまで再現すれば「ドーパミン創発の検証」として説得力が出るか。
