## 2. 観測された結果と違和感（32 persona × 100 step）

### 2.1 仮説検証の結果

- **仮説（ストレス × 大逆転認知 → 脳汁爆発）は外れた**
  - 高 stress 群（stress_load > 0.6）÷ 低 stress 群（< 0.4）の Δarousal 平均比が **1.3 倍に届かず**
- **Top 5 全員が「ボーナス当選（chain_start）」の瞬間** — 上乗せ・特化・確定演出が一度も Top 5 入りせず
- **追い詰められ度 0.16 の女子大生でも +30.2 の脳汁ピーク**を観測
- **首位は退職前男性**（sensory_gating 0.65-0.75、stress_load 0.4-0.55）
- 観察された傾向: **「感覚の鋭敏さ × 入り口イベント」**寄り

### 2.2 統計の罠の疑い

- chain_start の出現頻度が圧倒的多数 → 「最大値」が chain_start から出やすいだけかも
- 単 step 期待 Δarousal は理論的には kakutei (1.20) > tokka (1.00) > uwanose (0.60) > chain_start (0.40)
- だが Top 5 抽出は trigger を無視した降順なので、出現回数の多い chain_start が支配する

## 3. 実機経験から見る v4.2 の歪み（壁打ちで判明）

### 3.1 機種別「ボーナス当選の興奮度差」が表現されていない

実機の質感:
- ジャグラーの単発ペカ・ART初当たり・GOD 当選は **興奮度が桁違い**
- 普通の台のボーナス：軽い、頻発、「あ、入った」程度
- ミリオンゴッドのボーナス：稀、爆発、「神様！」
- 爆裂機 ART 突入：終わらない期待感

v4.2 の現状: arousal の `22.0 * hit_signal` は全機種同じ。upset_recognition も chain_start なら一律 0.40。
→ **機種ごとの「初当たりの重み」が全然違うのに、シミュレーションでは平等**。

### 3.2 上乗せ・特化ゾーンの「入りやすさ × 重み」差が表現されていない

実機の質感:
- **軽い上乗せ**：50G・100G、頻発、「ちょい乗り」程度の興奮
- **重い上乗せ**：500G〜、稀、「うわ、終わらない」
- **軽い特化**：CZ、頻発、入っても期待値低
- **重い特化**：GOD 凱旋・無想転生・リプレイハズレ目→特化、数万Gに 1 回、入れば終日

v4.2 の現状:
- 上乗せ 50/100/150/200G を一様乱数で選ぶだけ、出現確率も同じ
- 特化ゾーン：1 種類、`tokka_mult` で機種別調整するも種類差なし
- 確定演出：1 種類、`kakutei_mult` で機種別調整するも種類差なし
→ **「軽い・重い」の二相構造が無く、レアイベントの感動が消えている**。

### 3.3 stress_load が完全に固定値（step で動かない）

```python
# 現状: session 開始時に計算、100 step 通して同じ値
stress_load = 0.30 * persona.base_stress + 0.30 * borrow_burden + ...
```

実機の質感:
- 負け続けて cash が減るほど追い詰められる（stress 上がる）
- ボーナス当選で一旦解放（stress 下がる）
- ハマり長引くと焦燥が積み重なる
- 閉店時間が迫って当たらないと stress 急上昇

→ **「俺自身のストレスが大きい時」を表現する手段がない**（仮説検証の土台が崩れたまま）。

### 3.4 sensory_gating の命名と動作が逆

```python
raw_da *= persona.sensory_gating  # 高いほど Δarousal 増幅
```

- 命名は gating（抑制） だが、動作は **全感情の倍率**（増幅）
- Δdespair にも `(0.7 + base_despair / 100.0)` で似た形で効く
- 結果：「感覚鋭敏さの影響示唆」が観測されたが、これは sensory_gating の効果をそのまま見てるだけで、「**鋭敏さが大逆転認知に特に効く**」を示せていない

### 3.5 Top 5 抽出が物語を作れない

- 全員 chain_start に偏る → 動画的に **5 つとも同じシーン**で単調
- 上乗せ・特化・確定演出の **稀有な瞬間**を見せられない
- 「観察された傾向」として動画で押すには、Top 5 の構成自体が物足りない

### 3.6 ユーザー自身を投影できない

persona カテゴリで base_stress 範囲が固定。横塚自身の「今日のストレス」を入力して試す手段がない。
→ ハッカソン的にも「自分で試せるデモ」があると刺さる可能性があるが、設計に組み込まれていない。

## 4. 質問（変数優先度の洗い出し）

### Q1. 機種別「ボーナス当選の興奮度倍率」をどう設計すべき？

- arousal 更新式 `22.0 * hit_signal` を機種別に変えるべき？
- もしくは upset_recognition の chain_start 重み（現 0.40）を機種別パラメータ化？
- 提案: `mt.bonus_excitement_multiplier`（純A 0.5 / 4.5号機 1.0 / 2010s ART 1.5 / GOD 2.5 等）を導入し、`hit_signal * mt.bonus_excitement_multiplier` にする案。妥当？

### Q2. 上乗せ・特化ゾーンの「軽い・重い」二相構造をどう数値化すべき？

提案: trigger_event を細分化:
- `uwanose_light`（50-150G、出現率 高、重み 0.4）
- `uwanose_heavy`（500-1000G、出現率 低、重み 1.2）
- `tokka_light`（CZ系、出現率 0.04、重み 0.6）
- `tokka_heavy`（GOD 凱旋系、出現率 0.005、重み 2.0）
- `kakutei_engi`（無想転生・リプレイハズレ目、出現率 0.001、重み 3.0）

機種ごとに**どのレア度がどの確率で出るか**のテーブルを設計したい。妥当な確率設計の指針があれば。

### Q3. stress_load の動的更新式

提案ドラフト:
```python
delta_stress = (
    + 0.020 * loss_ratio_increase     # cash 減少加速
    + 0.015 * miss_streak_signal      # ハマり進行
    + 0.025 * cash_low                # 残金少
    + 0.030 * time_pressure_signal    # 閉店間際
    - 0.040 * chain_just_started      # ボーナスで一旦解放
    - 0.020 * gain_signal             # 勝ちで安心
)
stress_load = clip(stress_load + delta_stress, baseline_stress, 1.0)
```

- baseline_stress = session 開始時の固定値（下限）として残す
- 上限 1.0 まで動的に上下
- これで妥当？ 別の項を入れるべき？ 重みの感覚は？

### Q4. sensory_gating の命名と効果の整理

選択肢:
- **A. 命名だけ変える**：`sensory_amplitude`（感覚増幅）にリネーム、効果は維持
- **B. 効果を分離する**：gating は「全体抑制（dissociation 連動）」、爆発項への寄与は別属性 `dopamine_sensitivity` に切り出す
- **C. 削除する**：感情の個人差は base_arousal / base_despair / base_stress だけで表現

ハッカソン的にどれを推す？

### Q5. Top 5 抽出を trigger 種別ごとに分散させるべきか

提案: 「kakutei 最大 1 件、tokka_heavy 最大 1 件、uwanose_heavy 最大 1 件、long chain 最大 1 件、chain_start 最大 1 件」を 1 件ずつ拾う。
- メリット: 動画として上乗せ・特化・確定演出が見える
- デメリット: 「最大値の客観抽出」ではなくなる、評価バイアスが入る

ハッカソン提出文章では「**観察された傾向を 5 つの典型シーンで提示**」と位置づけるが、研究的には不純。許容範囲？
