## 3. 実機経験者からの修正提案（v4.1 ドラフト）

### 3.1 機種を 4 分類に拡張（沖スロ追加）

v4 は 3 機種（AT / Hokuto / ART）。これを 4 機種に。

| ID | 機種分類 | 代表機種 | キャラ |
|----|---------|---------|--------|
| 0 | 2000s 爆裂AT | ミリオンゴッド・獣王 | ギャンブル性極大、連荘薄い、爆発的払戻 |
| 1 | 4.5号機 | 北斗の拳・吉宗・番長 | ギャンブル性とゲーム性両立、ユーザーの主戦場 |
| 2 | 2010s 爆裂ART | 鬼武者・サラリーマン番長・秘宝伝 | 入口狭い、入れば終日 |
| 3 | マイルド純A・沖スロ | ジャグラー・大花火・ハナハナ | 軽い当たり頻発、連荘なし、未練打ち |

**割り切り**: 同分類内のバリエーション（北斗 vs 吉宗 vs 番長）は MVP では実装しない。「**世代の代表機種1つの典型値**」で扱う。提出文章に「実機再現でなく世代差の質感観察」と明記する前提。

### 3.2 機械割の現実化（v4 の最大の欠陥）

**v4 の数値の問題**:

```
GOD（v4 仮値）: p_initial_hit 0.04 × payout_mean 5500
= 期待値 220円/step（連荘抜き、連荘込みでも 296円）
= 機械割 約30%
```

これでは **設定1未満でほぼ全 run 大負け確定**。**興奮ピークが観測不能**。
ユーザーから「大負けシミュレーションにしかならないんじゃ？」という違和感。

実機 GOD のリアル:
- 当たり時 **平均1000枚（約20000円相当）**、稀に **5000枚（10万円超）**
- ハマるときは深いが、爆発時は爆発
- 機械割 設定6で 110-120%

**v4.1 修正案**:

| パラメータ | v4 | **v4.1 案** | 理由 |
|----------|-----|------------|------|
| GOD `p_initial_hit` | 0.04 | **0.025** | 確率は重く |
| GOD `payout_mean` | 5500 | **15000** | 1回の出玉を爆発的に |
| GOD `payout_std` | 3000 | **12000** | ばらつき大、たまの大爆発 |
| GOD `continue_prob` | 0.20 | **0.30** | 連荘で爆発を伸ばす |
| **payout 分布** | normal | **log-normal** | ハズレ多いが爆発もある |

**全機種の機械割を実機相当に再調整**:

| 機種 | v4 機械割（推定） | **v4.1 目標** |
|------|----------------|--------------|
| GOD系 | 約30% | **95-105%**（ばらつき大） |
| 4.5号機 | 約43% | **100-110%**（中庸） |
| 2010s ART | 約55% | **95-105%**（入れば伸びる） |
| 沖スロ | 約50% | **97-103%**（マイルド） |

### 3.3 感情を 3 軸に拡張

v4 は arousal + despair の 2 軸。ユーザーから「**負けすぎて麻痺してきて気持ちよくなる、悟り境地もある**」という指摘。

**追加軸**: `dissociation_level`（解離度、0-100）

|状態 | 表現 |
|-----|------|
| 麻痺・気持ちよくなる | 高 dissociation × 中 despair |
| 観察モード | 高 dissociation × 低 arousal × 低 despair |
| 悟り境地 | 高 dissociation × 長時間滞在 × 高 loss_ratio |

dissociation が高くなると arousal/despair の rule 更新値が**抑制**される（数値的に「感情が消える」を表現）。

更新式案:

```python
delta_dissociation = (
    + 0.8 * loss_ratio
    + 0.5 * miss_streak / 20  # ハマり長
    - 0.3 * sensory_gating_factor
    + 0.4 * (time_in_hall / 30)
)
dissociation_level = clip(dissociation_level + delta_dissociation, 0, 100)

# arousal/despair 更新値の抑制
suppression = 1.0 - (dissociation_level / 200)  # 0-100 → 1.0-0.5 の抑制係数
delta_arousal *= suppression
delta_despair *= suppression
```

→ **これで妥当？ 別の式の方が良い？**

### 3.4 persona 属性追加

ユーザーの実体験で「**焦り**」「**罪悪感**」が極めて重要だが、v4 で表現できてない。

#### `time_pressure`（0-1）
時間制約の度合い。閉店間際、バイト前、子供迎え等。

| 状況 | 値 |
|------|---|
| 不労所得・リタイア | 0.0 |
| 主婦（子供迎え） | 0.5 |
| 中年現役（バイト前・仕事サボり） | 0.7 |
| 閉店間際 | 1.0 |

#### `target_leave_step`（int）
離脱予定 step。`time_pressure × current_step / target_leave_step` で焦り信号生成。

#### `social_commitment_density`（0-1）
社会的約束の密度。**親父の手術中、親友の結婚式当日、葬式に行く前等の極端な状況**を含む。

| 状況 | 値 |
|------|---|
| 約束ゼロ（無職、独身、リタイア） | 0.0 |
| 通常（仕事・家族あり） | 0.3-0.5 |
| 重い約束（子供発表会、友人の結婚式） | 0.7-0.9 |
| **極端**（親の手術中、葬式当日、就活面接当日） | **1.0** |

これは実体験ベース。「親父の手術中に打ってた」「友達との約束をサボって連チャン優先した」のリアル。

#### `initial_cash` をカテゴリ別に
v4 では一律 20000円。これを persona カテゴリ別に:

| カテゴリ | initial_cash 案 |
|---------|---------------|
| 不労所得・地主 | 50000-100000 |
| 中年現役男性 | 20000-40000 |
| 依存症末期 | 10000-30000（借金プレイ） |
| 主婦 | 10000-20000 |
| 女子大生 | 5000-10000 |

### 3.5 感情更新式に項追加

#### arousal: 焦燥項

```python
# 残り時間が迫って当たらないと arousal が焦燥として上がる
time_pressure_signal = persona.time_pressure * (current_step / persona.target_leave_step)
delta_arousal += 8.0 * time_pressure_signal * miss_streak_signal
```

#### despair: 罪悪感項

```python
# 約束を破って打ってる罪悪感
guilt_signal = persona.social_commitment_density * (current_step / persona.target_leave_step)
delta_despair += 6.0 * guilt_signal
```

→ **重み 8.0 / 6.0 妥当？ 既存の重み（22.0 hit, 16.0 loss など）とのバランスは？**

## 4. 質問（11 個）

### Q1. GOD の payout_mean 15000 / std 12000 / log-normal で「興奮ピーク」観測できる？
v4 の最大の欠陥（大負け確定）を解消したい。この修正で現実的な観測実験になる？

### Q2. 機械割 95-110% に揃えると、機種差が薄れない？
GOD と沖スロの違いを保つには、何の指標で差を出すべき？（payout_std の大小？ continue_prob の高低？）

### Q3. log-normal 分布の具体的なパラメータ
GOD の場合、`np.random.lognormal(mu=?, sigma=?)` の `mu`, `sigma` は何にすべき？平均15000・標準偏差12000を再現するには？

### Q4. dissociation_level の更新式案、妥当？
3.3 の式（loss_ratio + miss_streak - sensory_gating + time_in_hall）で「麻痺」が観測できそう？別の項を入れるべき？

### Q5. dissociation の suppression 係数（1.0 - dissociation/200）は妥当？
arousal/despair の更新値を半分まで抑制する形だが、もっと強い抑制（dissociation/100 で完全抑制）の方が良い？

### Q6. social_commitment_density × time_progress で罪悪感を表現する案、妥当？
独立軸 `guilt` にしないで despair 増幅項として実装する形。これで「親父の手術中にプレイ」「友人の結婚式当日」シナリオを再現できる？

### Q7. time_pressure と target_leave_step の組み合わせで「閉店間際の焦り」表現できる？
2010年以降のART台で「**あと1時間しかないのに半日かけても終わらない**」感覚を出したい。

### Q8. initial_cash カテゴリ別の数値感は妥当？
不労所得 5-10万、依存末期 1-3万、女子大生 0.5-1万 の配分。

### Q9. 4 機種の typical 数値の見直し（特にGODと4.5号機）
3.2 の修正値に加えて、各機種で「**こんな数値の方が世代の質感が出る**」提案があれば。

### Q10. v4.1 で**見落としてる観点**は？
ユーザーの実体験壁打ちで出た以下を反映済み:

これ以外に、**実装前に詰めておくべき観点**は？
